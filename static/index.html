Tribal

Tribal is a credential lifecycle management tool that keeps your team ahead of certificate expirations, API key rotations, and other time-sensitive secrets — with automated Slack reminders, full audit visibility, and infrastructure-as-code support.

What Tribal Tracks

  • Certificates — upload a PEM or CRT file and Tribal extracts the expiry date automatically, or point it at a live TLS endpoint
  • API Keys — track third-party API keys with rotation deadlines and generation instructions
  • SSH Keys — log SSH key expiry and who's responsible for rotation
  • Any credential — use the "Other" type for anything else that needs monitoring

Automated Reminders

Tribal sends Slack notifications to the owning team at configurable intervals before a credential expires (e.g. 30, 14, 7, 3, and 1 days). Each reminder includes the rotation instructions and a link to the secret manager, so the on-call engineer can act immediately without hunting for context.

For Administrators

  • Notification Settings — configure when reminders fire (days before expiry) and the time of day (UTC)
  • Admin Slack Webhook — receive admin-level alerts: overdue resources, deleted resources, and test confirmations
  • User Management — promote members to admin, remove access, and protect the account creator
  • Audit Log — every create, update, delete, certificate upload, login, API key event, and role change is logged with the acting user and timestamp
  • Soft Deletes — deleted resources are retained and recoverable from the Admin panel; no data is permanently lost without explicit action
  • Reports — download CSV exports for upcoming expirations or recent audit activity
  • API Access — issue API keys for programmatic access; all API actions appear in the audit log with the source noted

Terraform Provider

Manage Tribal resources alongside your other infrastructure-as-code using the official Terraform provider. Define credentials as code, track them in version control, and enforce rotation policy through your existing IaC pipelines.

  • tribal_resource — create and manage tracked credentials
  • tribal_team — manage team groupings for organizing resources
  • tribal_admin_settings — configure org-wide notification schedules and Slack webhooks

Configure the provider with your API key (generate one under Account → API Keys):

terraform {
  required_providers {
    tribal = {
      source  = "seaburr/tribal"
      version = "~> 0.1"
    }
  }
}

provider "tribal" {
  host    = "https://your-tribal-instance"  # or set TRIBAL_HOST env var
  api_key = var.tribal_api_key              # or set TRIBAL_API_KEY env var
}

resource "tribal_team" "platform" {
  name = "Platform"
}

resource "tribal_resource" "prod_cert" {
  name                    = "api.example.com TLS Certificate"
  dri                     = "infra@example.com"
  type                    = "Certificate"
  expiration_date         = "2026-09-15"
  purpose                 = "TLS certificate for api.example.com"
  generation_instructions = "Run certbot renew or use ACM auto-renewal"
  slack_webhook           = "https://hooks.slack.com/services/..."
}
