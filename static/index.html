Docs

Tribal helps teams stay on top of expiration dates and rotation processes for certificates, API keys, SSH keys, and other time-sensitive credentials.

The Problem

Expired certificates and rotated credentials cause outages. Teams discover expirations too late because the knowledge of what expires, when, and how to rotate it is scattered across emails, wikis, and individual memory.

What Tribal Does

  • Tracks expiration and rotation dates for any credential or certificate
  • Sends automated Slack reminders 30, 14, 7, and 3 days before expiration
  • Stores generation and rotation instructions so any team member can act
  • Parses public certificate files to automatically extract expiry dates
  • Provides a visual calendar to see what's expiring and when

Terraform Provider

Manage Tribal resources via the UI or alongside your other IaC resources using the official Terraform provider. Define credentials as code, keep them version-controlled, and let Terraform handle drift detection.

  • tribal_resource — create and manage tracked credentials (certificates, API keys, SSH keys, etc.)
  • tribal_admin_settings — configure org-wide notification schedules and Slack webhooks

Configure the provider with your API key (generate one under Admin → API Keys):

terraform {
  required_providers {
    tribal = {
      source = "registry.terraform.io/cberndt/tribal"
    }
  }
}

provider "tribal" {
  # host defaults to TRIBAL_HOST env var
  # api_key defaults to TRIBAL_API_KEY env var
}

resource "tribal_resource" "prod_cert" {
  name                    = "api.example.com TLS Certificate"
  dri                     = "infra@example.com"
  type                    = "Certificate"
  expiration_date         = "2026-09-15"
  purpose                 = "TLS certificate for api.example.com"
  generation_instructions = "Run certbot renew or use ACM auto-renewal"
  slack_webhook           = "https://hooks.slack.com/services/..."
}
